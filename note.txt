#### avant toute chose je vais regarder ce qu'on nous demande dans le sujet mandatory :

- afficher une invite lorsqu'on attend l'utilisateur (comme avec here_doc de pipex)
- avoir un histique (normalement on a pas besoin de le coder mais d'appeler des fonctions, fichier ou autre parametre deja existant et autoriser)
- Rechercher et lancer les executables grace au path ou au chemin complet (comme sur pipex)
- utiliser une variable global pour eviter que les signaux n'accedera pas au donnees principal de struture de donnees.
- pas gerer les guillemets non fermes et autre caracteres speciaux pas requis pour le sujet (backslash ou point virgule)
- gerer guillemet simple ['] qui empeche le shell d'interpreter les metacaractere quand c'est entre guillement
- gere les guillement double ["] qui empeche comme les simple sauf le signe dollar [$]

implémenter les redirections suivantes :
< devrait rediriger les entrées.
> devrait rediriger la sortie.
<< devrait recevoir un delemiteur puis li tout comme un here_doc de pipex
>> devrait rediriger la sortie en mode d’ajout sur le fichier

implémenter les pipes avec tout les tuyaux connecter (voir si on peut reprendre mon pipex meme s'il n'est pas tres jolie)
gerer les variables d'environnement [$variable]
gerer la sortie exit ou return utiliser [$?]

gerer les racourci ; ctrl-C, ctrl-D et ctrl-  qui devrait se comporter comme dans bash
en interactif ;
ctrl-C affiche une nouvelle invite sur une nouvelle ligne.
ctrl-D quitte le shell.
ctrl-\  ne fait rien


implementer les commandes suivant ;
- echo : avec l'option -n
- cd : avec avec chemin relatif ou absolu
- pwd : sans option
- export : sans options
- unset : sans options
- env : sans options ni arguments
- exit : sortie sans options

note importante sur les leak :
- la fonction readline() peut avoir des fuites de memoire qu'on nous demande pas forcement de regler

- on doit egalement gerer les wildcards [*]
- cest pas demander mais si on se chauffe on peut aussi essayer de gerer le tild [~] histoire de faire un concours de zizi
